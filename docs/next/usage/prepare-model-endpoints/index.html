Prepare model endpoints for scanning

To scan your own model with Certifai, you must provide an endpoint that matches the expected predict API specified here.

Guidance on using Docker to package your python code for use with Certifai is provided here.

#Create a model endpoint using Flask

For development purposes, you can run your model in a local flask application without using Docker.

This example uses the banking_loan_approval use case from the Reference Models repository.

  1. In another terminal, clone the Reference Models repository:

    git clone git@github.com:CognitiveScale/certifai-reference-models.git

  2. Go to the banking_loan_approval folder. The Makefile in the all subdirectory will let you build and run a model server that has endpoints for four model implementations of this use case.

    cd certifai-reference-models/banking_loan_approval/all

  3. Create and activate a new conda environment.

    conda create -n model-server python=3.6
    conda activate model-server

  4. Install the prerequisites.

    pip install cortex-python flask gunicorn
    pip install -r requirements.txt

  5. Train the models using make train-all-local. When this completes you should see output ending similar to the following.

    ```
    ...
    Saved: SVM
    0.7966666666666666
    model: models/german_credit_svm.pkl
    ```

  6. Run the server using make predict-all-local. You should see build output that ends similar to the following.

    ```
    - Running on http://0.0.0.0:5111/ (Press CTRL+C to quit)
    ```
    If you are asked to allow an external connection, click 'Deny' as the model only needs to be available locally.
    If you get an error about `Address already in use`, make sure that your previous model server is not still running (CTRL-C to exit).
